eXTree 2.1 开发帮助文档
Toone Corporation   Apollo   2006.05.15
动态树xml文件<tree ... />元素的属性有：

公共的：id, text, src, type, icon, openIcon, color
普通树节点：action, target
单选(radio)、多选(checkbox)树节点：value, checked, disabled, onchange

注意：在Version 2.1中新增了自定义属性功能，自定义属性的名称不能和这里的属性同名。
version 2.1 更新：
新增树节点对象方法
setColor(color)、getColor()： 设置/获取节点的文本颜色。
文本颜色默认是黑色("black")，由webFXTreeConfig.defaultColor属性定义。 setColor方法对disabled的节点也有效。 如：将节点设为红色 node.setColor("red"); 动态加载节点可以在xml文件中指定color属性。

树节点对象增加自定义属性： 这一特性主要是用于动态加载树，通过xml文件 指定节点的自定义属性，注意自定义属性名不能和已有的关键字同名。
如： <tree id="nodeId" text="nodeName" myAttrName="myAttrValue"/>
该节点加载后，可以通过 getNodeById("nodeId").myAttrName 获取值 "myAttrValue"。
静态节点可以直接支持自定义属性：node.myAttrName = "myAttrValue"。

move(parentNode, index) ： 将节点移动为指定节点的子节点。 如：nodeA.move(nodeB); 节点nodeA（及其子节点，如有的话）将移动成为节点nodeB的 最后一个子节点。 目前只能移动成为最后一个子节点，不能指定索引位置(即index参数目前无效)，这是一个缺陷。

onchange事件属性： 这个属性指向一个JavaScript方法，当鼠标单击radio、checkbox时， 会触发这个事件，执行指定的JS方法。 动态加载树在xml文件中指定这个属性。<tree onchange="doMyFunction"/>

增强了节点的expand()： 当调用节点的expand()方法时，自动展开其祖先节点。 动态加载节点的expand()方法还有些小问题，要 webFXTreeConfig.userPersistence=false。
version 2.0 新增方法：
新增树节点对象方法
setId(id)和getId()： 设置节点 的Id属性，将业务对象和树节点进行关联。 对于动态树，可以在xml文件中指定id值。id属性不是必要的，可以不设置。
setText(txt)：设置节点对象的显示名称，方便不刷新整棵树而改变树节点名称。

其它：
getNodeById(id)：这是全局方法， 通过id获取树节点对象。要和 setId() 方法配合使用。

动态树：在配置文件中 <tree /> 元素增加type属性，指定这个节点的类型， 可以取值normal, check, radio三者之一（注意：区分大小写，只能是小写）， 分别表示“一般类型”、“checkbox类型”、“radiobutton类型”。 当不指定 type属性时，节点的类型与它的父节点类型相同。
树属性定义对象：
webFXTreeConfig : 定义了树的一些属性.
  这个对象在xtree.js文件的开头定义，可以对其属性进行修改，个性化自己的树。
增加的方法/属性：
方法：setImagePath(path)：设置eXtree所用的图片路径，如：webFXTreeConfig.setImagePath("/extree/images");
cascadeCheck：boolean, 定义CheckBox树是否级联选择
树根对象：
WebFXTree : 树的根节点对象.任何的树都可以使用它作为根节点.
   构造函数: WebFXTree(sText, sAction, sBehavior, sIcon, sOpenIcon)

WebFXLoadTree: 树的根节点对象, 同WebFXTree的区别是它的子结点在sXmlSrc指定,展开这个节点时才加载.
   构造函数: WebFXLoadTree(sText, sXmlSrc, sAction, sBehavior, sIcon, sOpenIcon)

WebFXCheckBoxLoadTree: 动态加载带CheckBox子结点的树根对象。
WebFXCheckBoxLoadTree(sText, sXmlSrc, sAction, sBehavior, sIcon, sOpenIcon)

WebFXRadioLoadTree: 动态加载带Radio子结点的树根对象。
WebFXRadioLoadTree(sText, sXmlSrc, sAction, sBehavior, sIcon, sOpenIcon)
节点基类：
WebFXTreeItem : 树中的节点对象.
  构造函数: WebFXTreeItem(sText, sAction, eParent, sIcon, sOpenIcon)

WebFXLoadTreeItem: 动态加载子节点的节点对象.
  构造函数: WebFXLoadTreeItem(sText, sXmlSrc, sAction, eParent, sIcon, sOpenIcon)

注：红色字体部分表示和1.0不同.

建议：有大量节点时eXtree的构造速度相当快，但是显示时（document.write(tree)）比较慢，其中控件树最慢。所以在有大量的节点时，首先建议大家使用动态树，其次是在控件树中尽量使用普通节点对象（比如在CheckBox树中的非叶子节点）。

largeCheckBoxTree.html 是测试速度页面，大家可以修改此页面，进行测试。

Map 类：
定义在map.js文件中，类似于Java中的Map对象。EXtree中CheckBox树使用到了该对象。
构造函数：Map()。
方法：

// 在Map中放入键值对。
put(key, object)

// 删除Map中键值key或者索引位置index对应的值。
// 参数 keyOrIndex 可以是键值，也可以是 index 索引。
// 返回删除的对象句柄，没有删除任何值时返回null。
remove(keyOrIndex)

// 获取Map中键值key或者索引位置index对应的值。
// 参数 keyOrIndex 可以是键值，也可以是 index 索引。
// 没有对应的值时返回null。
get(keyOrIndex)

// 获取Map中所有的值。
// 返回的是数组。
values()
